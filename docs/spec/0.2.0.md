# RepoGraph Protocol v0.2.0

## Changes from v0.1.0

### New optional config files

| File | Purpose |
|------|---------|
| `api.yml` | API endpoints mapped to modules |
| `database.yml` | Database entities mapped to modules |
| `risk.yml` | Path-based risk levels |
| `ownership.yml` | Team/module ownership |
| `glossary.yml` | Domain terminology |

These files are included as stubs in `repograph init` and validated when present.

### Analyzer improvements

- **Roslyn C# analyzer** (when .NET SDK and monorepo `tools/repograph-roslyn` are available): accurate `using` resolution, class symbols, API endpoints
- Falls back to heuristic regex analyzer when Roslyn is unavailable (e.g. npm global install)

### CLI

- `repograph diff --base main --head HEAD` — compare committed graphs between git refs
- Fails when new architecture violations are introduced vs base

## Migration from v0.1.0

1. Update `protocol.version` in `project.yml` to `0.2.0`
2. Run `repograph init --force` to add stub files (or copy stubs from [templates package](../../packages/templates/src/index.ts))
3. Re-run `repograph scan` to refresh `generated/graph.json`
4. Commit `graph.json` to git if using `repograph diff` in CI

## Backward compatibility

v0.2.0 CLI accepts v0.1.0 config files. New files are optional.
