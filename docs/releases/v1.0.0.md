# v1.0.0 — First public release

RepoGraph Protocol is ready for adoption: a machine-readable `.repograph` layer for your repo, a published CLI, CI integration, three examples, and release notes for every version.

## Install

```bash
npm install -g @repographprotocol/cli@0.2.1
repograph --version
```

## Highlights

### Protocol and CLI

- `.repograph` config: modules, layers, rules, tests, AI guidance
- Protocol v0.2 optional files: `api.yml`, `database.yml`, `risk.yml`, `ownership.yml`, `glossary.yml`
- Commands: `init`, `validate`, `scan`, `sync`, `check`, `diff`, `explain`, `impact`, `export`, `prompt`

### Trust and governance (Phase A)

- **Ownership** — two-way sync between `CODEOWNERS` and `.repograph/ownership.yml` via `repograph sync`
- **Risk** — signals-based `risk.yml` (path, file type, graph centrality, optional git churn)
- **Glossary** — suggested terms from code identifiers and docs

### Integrations

- **npm**: `@repographprotocol/cli`
- **GitHub Action**: `baasith6/RepoGraph-Protocol/action@v0.2.1`
- **MCP server** (monorepo): graph, explain, violations, impact, scan tools
- **VS Code extension** (monorepo): module tree + violations panel
